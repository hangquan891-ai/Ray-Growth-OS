# Ray Growth OS X Helper

这是 Ray Growth OS 的本地 Chrome 插件 MVP，用来补上免费版的 X 反馈闭环。

## 它做什么

1. 你从 App 打开原帖去 X 回复时，插件自动读取并关联当前队列条目，同时同步你的 X 用户名。
2. 插件只跟踪你实际打开并准备回复的条目，不要求预先批量读取队列。
3. 你发出回复后，插件会尝试在 X 页面识别你自己的回复链接。
4. 插件扫描你的回复链接下的公开互动计数，并把“有回复 / 无回复 / 被转发”等反馈回写到 App。

## 正确使用方式

1. 打开 Chrome 的 `chrome://extensions/`。
2. 打开右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择这个目录：`extension/ray-growth-os-x-helper`。
5. 项目或插件代码更新后，在扩展卡片上点一次“重新加载”，并刷新已经打开的 App 和 X 页面。
6. 打开 Ray Growth OS，并保持页面开着。在 App 设置里保存公开 X 主页，或者在插件里保存一次 X 用户名（`@` 后面的用户名，不是展示名）。插件弹窗不需要保持打开。
7. 必须从 App 队列里的“复制回复并打开原帖”进入 X。这个动作会自动读取并关联当前条目，同时保存本次准备采用的回复话术。
8. 在 X 发布回复，等自己的回复显示在当前原帖对话页。插件会自动保存回复链接并回写 App；状态会变为“已回复”。
9. 如果没有自动回写，停留在这个原帖对话页，打开插件点“找回当前页回复并回写”。这个按钮既能补抓漏掉的回复，也会立即尝试回写 App。
10. 过一段时间后再点“巡检已记录的回复链接”，检查这些回复后来是否收到公开回复或引用/转发。

“巡检已记录的回复链接”不会在 X 上反向搜索一次从未保存过的回复。它只处理已经有 `replyUrl` 的队列条目；漏抓时应先用“找回当前页回复并回写”。“手动回写暂存反馈”只用于插件已经识别到结果、但 App 当时没有打开或没有成功匹配的情况。

互动队列和反馈现在通过 App 的本机 SQLite 接口读写，不再依赖某一个浏览器里的工作台 `localStorage`。插件只在浏览器本地保存已跟踪条目和待回写反馈。更新项目后，请在 `chrome://extensions/` 中点击本插件的“重新加载”，并刷新已打开的 App/X 页面。

从 0.1.8 开始，插件回写会携带数据库版本；如果 App 页面同时更新，插件会重新读取并最多重试 5 次，不会用旧队列覆盖新数据。从 0.1.9 开始，不再需要手动读取队列；从 App 打开原帖时会自动关联当前条目。

## 当前限制

- 它依赖 X 页面的公开 DOM。X 改版后可能需要适配。
- 它不会自动替你发评论，只记录你已经发出去的公开回复。
- 它不读取私信、不绕过登录、不调用 X 付费 API。
- 找回漏掉的回复时，X 当前原帖对话页必须已经渲染出你自己的公开回复；如果页面上看不到，插件也无法从 DOM 识别。
