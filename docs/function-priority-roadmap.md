# 功能优先路线图

更新时间：2026-07-06

当前判断：页面视觉已经够用，后续优先级从 UI 精修切换到核心功能闭环。目标是让 Ray Growth OS 变成每天能打开、导入信号、评分、处理、复盘的工作台。

## 数据库策略

前期不需要服务端数据库，也不需要账号登录。

原因：

- 当前产品先服务 Ray 自己，数据只需要在本机浏览器保存。
- 还没有多用户、多设备同步、团队协作、付费空间这些需求。
- 上数据库会引入登录、权限、部署、备份、隐私和迁移成本，会拖慢核心功能验证。

推荐策略：

1. 第一阶段使用 `localStorage` 保存核心状态。
2. 如果信号数量明显变多，再升级到浏览器 `IndexedDB`。
3. 等产品要对外开放、需要登录和跨设备同步时，再迁移到服务端数据库。
4. 从一开始抽象一个 `StorageProvider`，避免以后迁移时重写业务逻辑。

当前不要存到本地的内容：

- OpenAI API Key
- xAI API Key
- X cookie / token
- 任何账号密码

这些如果需要，只能放 `.env.local` 或未来服务端加密存储。

## 功能拆分和优先级

### P0：本地持久化和统一数据结构

状态：进行中。已完成本地自动保存第一层，剩余结构化 Signal、JSON 备份/恢复、状态/draft 存储。

目标：刷新页面后，账号定位、关键词、导入的 Grok 信号、评分结果、草稿和处理状态都不丢。

为什么排第一：如果不持久化，Grok 导入一次、刷新就没了，无法形成真实日常工作流。

要做：

- 定义统一 `Signal` 数据结构。
- 把当前 `leadInput` 从纯 textarea 文本升级为结构化 signal 列表。
- 用 `localStorage` 保存：账号配置、关键词、signals、drafts、状态。
- 保留导入/导出 JSON，防止本地数据丢失。

### P1：Signal Inbox 导入箱

状态：已完成第一版。已完成结构化 Signal 工具层、Grok 导入预览、手动/CSV 导入、URL/文本去重、本地 signals 保存、JSON 备份/恢复。后续如有需要再补更复杂的导入字段映射。

目标：把 Grok 结果、手动文本、CSV 都统一导入成 `Signal[]`。

要做：

- Grok 粘贴结果解析。
- CSV / 文本导入。
- 去重：优先按 URL，其次按 source + sourceId，再其次按 text hash。
- 导入预览：展示解析到多少条、跳过多少条、重复多少条。

### P2：LLM 评分器

状态：核心能力，P1 后优先做。

目标：从“关键词命中加分”升级为“语义判断是否值得互动”。

实现策略：规则预筛 + LLM 精排。

要做：

- 后端新增 `POST /api/score`。
- `.env.local` 配置 `OPENAI_API_KEY` 和 `OPENAI_SCORE_MODEL`。
- 前端新增“AI 重新评分”按钮。
- 只对当前队列前 10-20 条执行，控制成本和速度。
- 失败时保留本地规则评分。

评分维度：

- 目标用户匹配度：25 分。
- 痛点强度：25 分。
- 可回复价值：20 分。
- 内容延展潜力：20 分。
- 互动时机和风险：10 分。

### P3：执行状态和每日队列

状态：P2 后做。

目标：让结果不只是列表，而是每天可处理的任务队列。

要做：

- 状态按钮：未处理、已回复、已引用、保存选题、跳过。
- 筛选：只看未处理、只看立即互动、只看保存选题。
- 每日队列：今天最值得处理的 5-10 条。
- 处理完成后从主队列降权或隐藏。

### P4：LLM 草稿生成器

状态：P3 后做。

目标：让回复、引用、选题更像 Ray 本人的表达，而不是模板句。

要做：

- 接入模型生成回复、引用、长帖选题。
- 支持风格选项：稳健版、犀利版、故事版、教程版。
- 使用账号定位、内容支柱、历史表达作为上下文。
- 保留本地模板草稿作为兜底。

### P5：反馈记录和复盘

状态：P4 后做。

目标：让系统知道哪些互动真的有效，反过来优化评分。

要做：

- 手动记录结果：是否收到回复、点赞、关注、私信、转化。
- 每周复盘：哪些关键词、话题、回复角度更有效。
- 给 scoring rubric 增加反馈权重。

### P6：真实数据源连接器

状态：功能闭环跑通后做。

目标：减少手动复制粘贴。

顺序：

1. xAI / Grok API 或中转接口。
2. GitHub Search API 备用。
3. X API Recent Search 视成本和权限再决定。
4. Reddit 后置验证。

注意：Grok 网页自动化已经实测会被 Cloudflare 拦截，不作为默认路线。

### P7：视觉统一和公开展示

状态：功能跑通后再做。

目标：当产品真的能每天用了，再回来做完整视觉统一、截图、公开演示和发帖素材。

要做：

- 把剩余白色模块统一成暗色玻璃风。
- 移动端视觉 QA。
- 做一个无需登录的 demo 示例数据。
- 写 X 长帖介绍构建过程和使用效果。
